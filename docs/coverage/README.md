# Coverage snapshot

HTML report generated by `pnpm coverage` (Vitest v8 provider).

## View it

**Online** (renders the HTML directly via a free GitHub proxy — no clone needed):
[`docs/coverage/index.html` via htmlpreview.github.io](https://htmlpreview.github.io/?https://github.com/DavidELBAZpro/voxfit-post-call-decision-engine/blob/main/docs/coverage/index.html)

**Locally:**
```sh
pnpm coverage
open coverage/lcov-report/index.html  # macOS
xdg-open coverage/lcov-report/index.html  # Linux
```

## Regenerate this snapshot

```sh
pnpm coverage
rm -rf docs/coverage
cp -r coverage/lcov-report docs/coverage
git add docs/coverage && git commit -m "chore: refresh coverage snapshot"
```

## Why a snapshot?

`coverage/` is gitignored (it's a build artifact). But for the reviewer, browsing the
HTML report on GitHub is much easier than cloning, installing, and running the tests.
So we commit a one-time snapshot in `docs/coverage/`. It can go stale between runs —
treat the live `pnpm coverage` output as the source of truth.
